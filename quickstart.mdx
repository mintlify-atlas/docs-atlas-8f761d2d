---
title: "Quickstart"
description: "Get Squiggle running locally in under 5 minutes."
---

<Note>
  The word bank is stored server-side in Firestore and is never sent to the client. This prevents players from cheating by reading the source or network traffic.
</Note>

<Steps>
  <Step title="Clone the repo">
    Clone the repository and install dependencies:

    ```bash
    git clone https://github.com/thor-op/squiggle.git && cd squiggle && npm install
    ```
  </Step>

  <Step title="Set up Firebase">
    Squiggle requires a Firebase project with Firestore enabled. The three steps are:

    1. Create a project at [console.firebase.google.com](https://console.firebase.google.com)
    2. Enable **Firestore** in Native mode
    3. Generate an Admin SDK private key from **Project Settings → Service Accounts → Generate new private key**

    See [Firebase setup](/setup/firebase) for the full walkthrough.
  </Step>

  <Step title="Configure environment variables">
    Copy the example env file and fill in your values:

    ```bash
    cp .env.example .env.local
    ```

    You'll need both the client config values and the Admin SDK credentials. See [Environment variables](/setup/environment-variables) for the complete reference.
  </Step>

  <Step title="Deploy Firestore rules">
    Deploy the security rules that block client access to the word bank:

    ```bash
    npx firebase login
    npx firebase use your-project-id
    npx firebase deploy --only firestore:rules
    ```

    Replace `your-project-id` with your actual Firebase project ID.
  </Step>

  <Step title="Seed the word bank">
    Words live in Firestore, not in the client bundle. You need to seed them before starting a game.

    See [Word bank](/setup/word-bank) for instructions on copying the seed script and running it.
  </Step>

  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser.
  </Step>
</Steps>
