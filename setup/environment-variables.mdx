---
title: "Environment variables"
description: "Reference for all environment variables used by Squiggle."
---

Copy `.env.example` to `.env.local` and fill in your values before running the app.

```bash
cp .env.example .env.local
```

<Warning>
  `FIREBASE_ADMIN_*` variables are server-only. They must never be prefixed with `NEXT_PUBLIC_` — doing so would expose your service account credentials to every browser that loads the app.
</Warning>

## Client variables

These are prefixed with `NEXT_PUBLIC_` and are safe to expose to the browser. They come from your Firebase web app config.

<ParamField path="NEXT_PUBLIC_FIREBASE_API_KEY" type="string" required>
  Your Firebase web API key. Found in **Project Settings → General → Your apps → Web app config**.
</ParamField>

<ParamField path="NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN" type="string" required>
  Your Firebase auth domain. Typically `your-project-id.firebaseapp.com`.
</ParamField>

<ParamField path="NEXT_PUBLIC_FIREBASE_PROJECT_ID" type="string" required>
  Your Firebase project ID. Used to identify your project across Firebase services.
</ParamField>

<ParamField path="NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET" type="string" required>
  Your Firebase Storage bucket. Typically `your-project-id.firebasestorage.app`.
</ParamField>

<ParamField path="NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID" type="string" required>
  Your Firebase Cloud Messaging sender ID.
</ParamField>

<ParamField path="NEXT_PUBLIC_FIREBASE_APP_ID" type="string" required>
  Your Firebase web app ID.
</ParamField>

<ParamField path="NEXT_PUBLIC_FIREBASE_DATABASE_URL" type="string">
  Your Firebase Realtime Database URL. Optional — Squiggle uses Firestore, not the Realtime Database.
</ParamField>

<ParamField path="NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID" type="string">
  Your Google Analytics measurement ID (e.g. `G-XXXXXXXXXX`). Optional — only needed if you enabled Analytics.
</ParamField>

<ParamField path="NEXT_PUBLIC_FIREBASE_FIRESTORE_DB" type="string">
  A custom Firestore database ID if you are using a named database. Leave blank to use the default database.
</ParamField>

## Server-only variables

These are used exclusively by the Firebase Admin SDK on the server. They come from the service account private key JSON you download from Firebase.

<ParamField path="FIREBASE_ADMIN_PROJECT_ID" type="string" required>
  Your Firebase project ID. Matches `NEXT_PUBLIC_FIREBASE_PROJECT_ID` in most cases.
</ParamField>

<ParamField path="FIREBASE_ADMIN_CLIENT_EMAIL" type="string" required>
  The service account email address. Looks like `firebase-adminsdk-xxxxx@your-project.iam.gserviceaccount.com`.
</ParamField>

<ParamField path="FIREBASE_ADMIN_PRIVATE_KEY" type="string" required>
  The RSA private key from the service account JSON. Wrap the entire value in double quotes and preserve the literal `\n` sequences — they are replaced with real newlines at runtime.
</ParamField>

## Example `.env.local`

```bash
# Firebase config — copy to .env.local and fill in your values
NEXT_PUBLIC_FIREBASE_API_KEY="your-api-key"
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN="your-project.firebaseapp.com"
NEXT_PUBLIC_FIREBASE_DATABASE_URL="https://your-project-default-rtdb.firebaseio.com"
NEXT_PUBLIC_FIREBASE_PROJECT_ID="your-project-id"
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET="your-project.firebasestorage.app"
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID="your-sender-id"
NEXT_PUBLIC_FIREBASE_APP_ID="your-app-id"
NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID="G-XXXXXXXXXX"
# Optional: custom Firestore database ID (leave blank for default)
NEXT_PUBLIC_FIREBASE_FIRESTORE_DB=""

# Firebase Admin SDK — server-side only (never exposed to browser)
# Get these from Firebase Console → Project Settings → Service Accounts → Generate new private key
FIREBASE_ADMIN_PROJECT_ID="your-project-id"
FIREBASE_ADMIN_CLIENT_EMAIL="firebase-adminsdk-xxxxx@your-project.iam.gserviceaccount.com"
FIREBASE_ADMIN_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
```
